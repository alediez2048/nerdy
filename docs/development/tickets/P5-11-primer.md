## What Is This Ticket?

P5-11 extends the **README.md** so a new developer can run the full pipeline and open the dashboard in under 5 minutes. One-command setup is an explicit requirement in the rubric. The README is the first thing a reviewer sees — it must make the system's value obvious immediately.

### Why It Matters

- Code Quality Requirements explicitly list "One-command setup (requirements.txt or package.json)" and "Concise README with setup and usage"
- Automatic deduction of -10 points for "Cannot run with provided instructions"
- **The Reviewer Is a User, Too** (Pillar 8): The submission is a product, the reviewer is the customer
- A great system that's hard to run will score worse than a good system that runs in one command
- The README frames every other deliverable — it's the table of contents for the entire project

## What This Ticket Must Accomplish

### Goal

Extend README.md so that a reviewer can clone the repo, set up the environment, run the pipeline, and open the dashboard in under 5 minutes — with a single command for setup and a single command for execution.

### Deliverables Checklist

#### A. README Sections (extend `README.md`)

- [ ] **Project Title + One-Line Description** — What this is in one sentence
- [ ] **Quick Start (< 5 minutes)**
  - One-command setup: `pip install -r requirements.txt` (or equivalent)
  - Environment configuration: API key setup (`.env` or `config.yaml`)
  - Run the pipeline: `python run_pipeline.py`
  - Open the dashboard: instructions to open the HTML dashboard
  - Expected output: what the user should see when it works
- [ ] **Architecture Overview**
  - High-level pipeline flow diagram (ASCII or reference to image)
  - Module structure with brief descriptions:
    - `generate/` — Ad copy generation from briefs
    - `evaluate/` — Dimension scoring, LLM-as-judge, aggregation
    - `iterate/` — Feedback loop, improvement strategies
    - `output/` — Formatting, export, visualization
    - `data/` — Brand knowledge base, reference ads, config
    - `tests/` — All test suites
    - `docs/` — Decision log, writeup, deliverables
  - Key design decisions (brief, link to decision log for details)
- [ ] **Usage**
  - Full pipeline run: `python run_pipeline.py`
  - Resume interrupted run: `python run_pipeline.py --resume`
  - Export ad library: `python output/export_ad_library.py`
  - Run tests: `pytest`
  - Open dashboard: specific instructions
- [ ] **Configuration**
  - `config.yaml` parameters: batch size, quality threshold, model routing, token budgets
  - Environment variables: API keys, model selection
  - Brief configuration: audience segments, campaign goals
- [ ] **Dashboard**
  - How to launch the dashboard
  - What each panel shows (brief descriptions of the 8 panels)
  - Screenshot or reference
- [ ] **Deliverables**
  - Link to decision log: `docs/deliverables/decisionlog.md`
  - Link to technical writeup: `docs/deliverables/writeup.md`
  - Link to demo video
  - Link to ad library export: `output/ad_library.json`, `output/ad_library.csv`
- [ ] **Testing**
  - How to run tests: `pytest`
  - Test coverage summary: what's tested
  - Expected test count (15+)
- [ ] **Limitations**
  - Known issues and failure modes
  - Gemini free tier rate limits and their impact
  - What the system doesn't do
